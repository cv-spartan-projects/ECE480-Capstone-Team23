Development

From concept to system

The project evolved through simulation, breadboard validation, PCB design, and full integration — sponsored by Texas Instruments to demonstrate the INA351A in a complete, portable application.

Phase 1
The Goal
Design a compact ECG signal acquisition and display system sponsored by Texas Instruments. The system must capture a real ECG signal, process it through an analog front end, and display the waveform in real time on an LCD — operating as a standalone, portable demo platform built around the INA351A instrumentation amplifier.
Phase 2
Component Selection
A weighted decision matrix evaluated key subsystems. The TI MSPM0C1106 was chosen for its TI ecosystem alignment, ultra-low power, and sufficient ADC resolution. An LCD module was selected over OLED/TFT for clarity, simple interfacing, and low cost. The INA351A was a fixed TI requirement as the primary device under demonstration.
Phase 3
Signal Chain Design
Full ECG signal chain designed: INA351A for instrumentation amplification (gain 20), followed by an OPA333 gain stage reaching ~1000 V/V total. A passive RLD connection stabilizes body potential. The system runs on a single 3.3 V supply with a 1.65 V virtual reference (VREF) buffered by an OPA333 — allowing the bipolar ECG signal to swing correctly without a negative supply rail.
Phase 4
0.6–33 Hz Passband
Sallen-Key HPF and LPF stages were tuned experimentally using swappable through-hole components on the breadboard, enabling rapid real-time frequency testing. Final values establish a passband that removes baseline drift and high-frequency noise while preserving the PQRST complex.
Phase 5
MCU + LCD Integration
Firmware developed for ADC sampling at ~250 Hz, real-time waveform rendering, and touchscreen control. A state machine enables navigation between the main menu, live mode, and simulated mode. ADC polling ensures each conversion completes before the sample is processed. A pause feature relieved SPI bandwidth pressure when switching screens.
Phase 6
KiCad Custom PCB
Two PCB variants designed in KiCad: one with a 3.5 mm AUX jack for gel electrodes, one with exposed copper fingertip pads — both using the same signal chain. Component placement follows signal flow with analog/power separation, close decoupling caps, and a full ground plane. Multiple iterations incorporated sponsor feedback. Due to fabrication timing, system validation was completed on breadboard/perfboard.
Phase 7
Enclosure & Final Build
Two enclosures developed: an acrylic open-display platform for transparent demos, and a 3D-printed enclosure with hexagonal ventilation for a product-like form factor. A branded NANUK 910 hard case provides rugged transport with MSU and TI logos. The final system captures and displays live ECG waveforms running fully from a USB power bank.
Challenges

Problems we solved

Six significant engineering challenges encountered and resolved over the course of development.

Hardware
Single-Supply Bipolar Signal
ECG signals are bipolar but the system runs on a single 3.3 V supply. Fixed by creating a 1.65 V virtual reference (VREF) and biasing the entire signal chain around it, allowing the ECG to swing above and below VREF without clipping.
Hardware
Filter Cutoff Tuning
The waveform would slowly taper flat — indicating an incorrect HPF cutoff. Swappable through-hole components on the breadboard allowed rapid experimental tuning to find the correct 0.6 Hz high-pass cutoff.
Hardware
VREF Stability
Multiple stages drawing current from the resistor-divider VREF caused voltage drift and signal instability. Adding an OPA333 buffer to the VREF node created a stable, low-impedance reference for the full signal chain.
Hardware
Electrode Interface
The circuit worked with a function generator but produced unstable real-human signals. Gel electrode pads were adopted for validation; copper fingertip pads were used in the final design for portability.
Firmware
ADC Conversion Failures
No ADC values were initially displayed on the LCD. Implementing ADC polling — having the CPU wait for each conversion to complete before processing — resolved the issue and ensured reliable sample capture.
Firmware
Touchscreen + Waveform Conflict
SPI bandwidth was fully consumed by waveform rendering, blocking touchscreen inputs. A waveform pause feature freed the SPI bus to process touch events and enable proper screen navigation.

How to Use This System

You scanned the QR code and landed here. This guide walks you through powering on and operating the ECG Signal Acquisition & Display System from scratch — no prior experience needed.

⚠️
Important: This is a demonstration and educational device only. It is not a medical device and is not intended for clinical diagnosis, treatment, or medical use of any kind.
Operating Modes
🔴 Live Mode — Real ECG
Requires the device, a charged USB power bank, and either gel electrode pads (3.5 mm AUX cable) or the exposed copper fingertip pads on the board. Works best with clean, dry fingertips and minimal motion.
⚫ Simulated Mode — No Electrodes
Built-in simulated ECG signal accessible directly from the LCD menu. No electrodes required. Use this to verify the display and system are functioning before testing with live signals.
Step-by-Step Guide
1
Power on the system
Connect the USB power bank to the system's USB input port. The LCD display should illuminate and show the main menu within a few seconds. If nothing appears, verify the power bank is charged and the cable is firmly seated.
2
Verify with simulated mode first
From the main menu, tap "Simulated Mode" on the touchscreen. A synthetic ECG waveform should begin scrolling across the display. This confirms the MCU, LCD, and firmware are functioning. Tap the screen to pause/resume and navigate back.
3
Connect electrodes (for live ECG)
For gel pads: plug the 3.5 mm cable into the AUX jack and attach pads to your wrists/ankle (Lead I placement). For fingertip pads: press both index fingers firmly and consistently onto the two exposed copper pads on the board.
4
Start live ECG mode
Tap "Live Mode" from the main menu. The display will begin rendering your ECG in real time. Hold still — motion creates noise. You should see a repeating PQRST waveform within a few seconds once your signal is cleanly captured.
5
Pause and navigate
Tap the screen while in waveform mode to pause the display. This frees the SPI bus so the touchscreen responds properly. From the paused state you can resume, or navigate back to the main menu to switch modes.
6
Power off
Navigate back to the main menu, then disconnect the USB power bank. No data is saved between sessions — the system starts fresh each power cycle. Store in the branded NANUK hard case for transport.
💡
Tip: Signal quality with fingertip pads depends on consistent contact pressure. If the waveform looks noisy, press more firmly and hold still. Gel electrodes provide a significantly more stable signal when available.
